《建筑能源智能管理与运营优化系统》项目落地实施报告书
一、 项目定位与核心价值

本项目响应“双碳”目标，聚焦建筑能源管理中“运维依赖经验、数据价值挖掘不足、分析手段单一”等痛点。通过**“本地边缘监测+云端专家诊断”**的协同架构，实现能耗数据的精准查询、能效自动分析、智能故障诊断及自动化报表生成。

二、 总体技术架构：云边协同双引擎

系统采用分级智能架构，确保数据安全、计算精准与深度洞察的完美结合：

边缘端（Local Edge）：部署 ForeseeAI Qwen3-4B (INT4)。负责本地数据隐私脱敏、自然语言转SQL（Text-to-SQL）、以及简单意图解析。

云端（Cloud Intelligence）：调用 阿里云百炼 (GLM-5 / Qwen3-Plus)。负责多模态视觉识别（巡检图）、深度运维推理及专业报告润色。

计算引擎（Logic Engine）：基于 Python/Pandas。负责100%准确的数学计算（COP、同比环比、异常筛选）。

知识中枢（RAG）：基于 RagFlow。存储设备手册与运维规程，消除大模型幻觉。

协议中台（Protocol）：采用 MCP (Model Context Protocol)。统一工具调用接口，实现模型与数据库的标准化解耦。

三、 落地实施详细步骤
第一阶段：数据底座建设（Data Foundation）

目标：构建规范化、结构化的能耗数据库。

操作要点：

环境搭建：安装 PostgreSQL，创建 building_energy 库。

表结构映射：按照提供的 Excel 宽表设计 energy_reports 表。

ETL 脚本编写：编写 Python 脚本，处理 Excel 中的时间格式（纠正 ########），将 1 年期的模拟数据导入数据库。

数据字典编制：定义 Data_Dictionary.md，明确字段含义与正常阈值，作为 RAG 的核心知识。

第二阶段：分析引擎与报表开发（Analysis & Reporting）

目标：实现“定量分析”的自动化。

操作要点：

逻辑封装：编写 EnergyAnalyzer 类，内置 calculate_cop()、get_daily_summary() 等函数。

算法设计：使用统计学方法（如 Z-Score）建立能耗异常识别模型。

报表自动化：利用 XlsxWriter 库，根据 SQL 查询结果自动生成带有 ECharts 风格图表和 Excel 表格的分析总结包。

第三阶段：本地与云端 AI 部署（AI Deployment）

目标：打通“人机对话”与“工具调用”链路。

操作要点：

本地部署：使用 llama-cpp-python 部署 ForeseeAI 4B。编写 System Prompt 引导其进行精准的任务分发（Router）。

云端接入：配置阿里云百炼 API 密钥。在控制台建立 RAG 知识库，上传《设备运维手册》。

MCP 接口开发：将 Python 编写的分析函数通过 MCP 协议暴露给大模型。

多模态配置：联调视觉接口，确保系统能接收巡检照片并关联对应的数据库记录。

第四阶段：Web 系统集成（System Integration）

目标：提供直观、顺畅的交互界面。

操作要点：

后端服务：使用 FastAPI 整合本地/云端模型流。提供 /chat、/query、/export 三大核心接口。

前端界面：

交互区：对话框式查询，支持文字与图片输入。

可视化区：集成 ECharts，根据模型生成的 JSON 动态渲染能耗折线图、能效饼图。

报表区：历史分析报告列表，支持一键预览与下载。

四、 核心功能场景演示流

多维度智能查询：

用户问：“查一下行政楼上周二下午的电耗峰值。”

系统：本地模型转 SQL -> 后端查库 -> 弹出数值并伴随该时段的趋势图。

能效诊断分析：

用户问：“为什么本月 Aral 楼的 COP 效率整体偏低？”

系统：云端 Agent 获取全月数据 -> 发现温差异常 -> 检索 RagFlow 手册 -> 回复：“分析发现温差长期小于 3℃，推测为冷凝器结垢，建议安排清洗。”

自动化报表生成：

用户问：“生成一份本季度的运行分析简报。”

系统：自动汇总数据、自动绘图、自动撰写 AI 专家建议 -> 生成 Excel 下载链接。

五、 项目亮点与竞争优势（答辩要点）

云边协同架构：本地轻量级模型保证了基础查询的极速响应和数据隐私；云端大模型保证了运维诊断的深度。

计算精度保障：坚决不让 LLM 算数学题。系统采用“模型提参数、代码算结果、模型写结论”的科学闭环。

多模态运维：支持巡检照片识别，实现了从“纯数字管理”到“视觉增强运维”的跨越。

工业扩展性：基于 MCP 协议开发，可无缝平滑接入中建八局“擎翼数字中枢”等实际工程平台。

六、 结论与落地建议

本项目已完成从“原始数据”到“智能架构”的闭环规划。建议团队立即按照 “导数 -> 算数（Python）-> 调模型（百炼/本地）-> 套壳（Vue）” 的优先级顺序推进开发。

该方案不仅满足了赛题的应用类要求，更在智能计算与智慧制造的深度结合上具有极强的创新示范意义。
