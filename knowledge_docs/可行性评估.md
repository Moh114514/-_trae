## 一、技术可行性 ✅ 高

### 1. 边缘端部署（ForeseeAI Qwen3-4B INT4）

| 项目 | 评估 |
|------|------|
| **可行性** | 高 |
| **硬件要求** | 最低 4GB 内存，推荐 8GB+；GPU 可选 |
| **部署方案** | llama-cpp-python 成熟稳定，支持 CUDA/Metal/CPU |
| **风险点** | 纯 CPU 推理延迟 2-5 秒 |

**注意**: ForeseeAI Qwen3-4B 需确认获取渠道，或改用开源 Qwen2.5-3B-GGUF。

## 五、风险与建议

### 高优先级风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| COP 计算公式准确性 | 高 | 与业务专家确认，编写单元测试 |
| 设备运维手册缺失 | 高 | 尽快收集或创建 |

### 建议

1. 优先完成数据底座建设（PostgreSQL + 表结构）
2. RAG 知识库可与 ETL 并行准备
3. 先打通单模型链路，再做多模型协同
4. 前端可用 Mock 数据并行开发
